Some notes the MLE

While the methods described here can be used on nearly any type of model, were going to focus on logistic regression because it has nice form, and is straightforward in its interpretation.

Before we get into gradient descent, I will quickly review maximum likelihood estimation (MLE). MLE is a mathematical formulation that chooses estimators of parameters based on the information provided by available data. It chooses estimates for the parameters by finding the estimates that produce the largest output value from the likelihood function (i.e.the largest evaluation of the likelihood means that those estimates were the most likely). One of the nice properties of the MLE is that it avoids both loss functions and prior distributions. However, you will also find it has limitations (but we wont get into those here).

The MLE can be easily summarized as follows.

Let:

  • \(\theta\) = The true, unknown distributional parameters
  • \(\hat\theta\) = The estimated parameters
  • \(\textbf{x}, x_i\) = A collection of observations, either a matrix and vectors, or a vector and elements (respectively)
  • \(f(x_{i} | \theta)\) = A probability distribution of \(x\), or, a conditional probability distribution of \(x\) given \(\theta\)
  • \(L(\theta | x), f_{n}(\textbf{x} | \theta)\) = The likelihood function

Then:

\[L(\theta | x) = f_{n}(\textbf{x} | \theta) = \prod_{i = 1}^n f(x_{i} | \theta) \]

And:

\[\hat\theta = \underset{\theta \in \Theta}{\operatorname{argmax}} L(\theta | x) \]

While acknowledging that a PDF is different from a PMF, for practical reasons, I will refer to them identically as PDF, since the rest of what I have to say will not depend of whether \(f(x_{i} | \theta)\) is a PDF or PMF.

Literature sometimes refers to a PDF as a likelihood. Semantically, this could be considered correct because the evaluation of a single point in a PDF is not a probability value (i.e.the area under the curve is the probability, and the probability for any given point on the curve is 0, since \(w \times h\), and \(w=0\) in the Riemann sum rectangle, for example), and instead, represents a measure of likelihood. However, I reject this verbiage because I consider it confusing. Instead, I prefer to think of the likelihood as something thats like a PDF, but is mathematically different. In the MLE definition above, the likelihood \(L(\theta | x)\) is the product of several PDFs, or rather a single PDF evaluated at different data points. Its no longer a probability distribution because it doesnt integrate to one (you could force it to be a PDF by adding a constant to the integration, but thats not worth it in most cases). For simplicity, we will refer to the likelihood using the definitions provided above.

Solving an MLE problem can take on many forms. The most common non-computational method is to find the first derivative and set it to zero, and solve for the parameters. In practice, this is either difficult or impossible, and so many numerical methods exist, including Nelder-mead (requires no knowledge of derivatives) and gradient decent (requires first order partial derivatives).

An example of such a derivation is shown below using an exponential likelihood.

\[L(\theta | x) = \prod_{i = 1}^n f(x_{i} | \theta)\]

\[ \ln(L(\theta | x)) = \ln \Big( \prod_{i = 1}^n f(x_{i} | \theta) \Big)\]

\[ \ln(L(\theta | x)) = \sum_{i = 1}^n \ln \Big( f(x_{i} | \theta) \Big)\]

Let \(f(x_{i} | \theta)\) be the exponential distribution, and let \(\theta = \lambda\), then:

\[ \ln(L(\lambda | x)) = \sum_{i = 1}^n \ln \Big( \lambda e^{-\lambda x_{i}} \Big) \]

\[ \ln(L(\lambda | x)) = n \ln(\lambda) + \sum_{i = 1}^n \Big( -\lambda x_{i} \Big) \]

\[ \ln(L(\lambda | x)) = n \ln(\lambda) - \lambda \Big( \sum_{i = 1}^n x_{i} \Big)\] \[ \frac{d}{d \lambda}\ln(L(\lambda | x)) = \frac{n}{\lambda} - \Big( \sum_{i = 1}^n x_{i} \Big)\] Set the derivative to zero

\[ \frac{d}{d \lambda}\ln(L(\lambda | x)) = 0 = \frac{n}{\lambda} - \Big( \sum_{i = 1}^n x_{i} \Big)\]

\[ \Big( \sum_{i = 1}^n x_{i} \Big) = \frac{n}{\lambda}\]

\[ \lambda_{MLE} = \frac{n}{\Big( \sum_{i = 1}^n x_{i} \Big)}\]

With this formula, we can determine the \(\lambda_{MLE}\) that maximizes the likelihood of the data we have observed. Another way to say this would be to say that given the data we observed, \(\lambda_{MLE}\) is the most likely estimate of the true, underlying distribution that we sampled our data from. For likelihood functions with more than one parameter, there is not always an easy derivation, and sometimes no closed form solution. In these cases, it is recommended to use numerical solvers such as Nelder-Mead or gradient descent.

On to Gradient Descent

The reason we introduced the MLE first is because it builds a basis for transitioning to machine learning terminology and techniques. For example, when the cost function is equivalent to a likelihood function, the outcome of performing gradient descent, or using Adam optimizers, or any other numerical solving method is the same: the parameters that maximize the likelihood of the underlying statistical distribution is obtained. However, if the cost function is something that doesnt belong to a family of statistical distributions, then this may not be true (and even in some instances, machine learning practitioners impose other constraints on the likelihood such that the solution obtained is not equivalent to the MLE for the same model/cost formulation). Often, machine learning textbooks leave out these probabilistic roots, which is why I include them here.

Definition and formulation of a gradient

To keep things simple, well talk about gradients in the context of machine learning rather than coordinate systems or vector spaces (as much as possible). Im also skipping over limits for the time being.

If youll remember from math, the first derivative (or first order derivative) of a function is the slope of the tangent line at a given point. In practical terms, this represents the rate of change in the function at that point. The first derivative tells us when a function is increasing or decreasing, which is very helpful when trying to find minima or maxima (as we saw in the MLE example).

So a gradient is just a vector of derivatives, specifically the partial derivatives for some function \(f(\textbf{x})\). In one dimension, the gradient is a vector with one element: the derivative. In multiple dimensions, the gradient is a vector with many elements that collectively represent a plane (or other higher dimensional representations). The gradient identifies the normal of the tangent plane (the equivalent of slope of the tangent line in 1D), and therefore can identify the direction of steepest ascent.

In one dimension, the derivative/gradient identify the direction of steepest ascent using the sign of the slope. For example, let \(f(x) = x^2\), the basic parabola, such that \(f'(x) = 2x\). If \(x=1\), the slope is positive, indicating that moving \(x\) in a positive direction will increase the function value (climb higher), and the value of the slope is indicating how fast the function is able to climb in that spot. If \(x=-1\), the interpretation is the same, but this time its indicating that the function will increase faster if \(x\) moves in a negative direction.

The same logic extends to a gradient vector. Each element of the vector indicates the direction of steepest ascent for the function, given all else is constant regarding the other elements (partial derivatives). Collectively, the gradient provides a clear path of ascension.

The natural mathematical formulation of derivatives and gradients provides the path of steepest ascent. This will be true for either concave up or concave down functions (for example, \(f(x) = x^2\) is concave up, \(f(x) = -x^2\) is concave down, but both of their derivatives indicate the direction to climb at any given point). Gradient ascent or gradient descent is a method of finding where the function value is its highest/lowest (known as a global minimum/maximum), as well as what the function parameter values are at that point. Thats the goal, however, its worth noting that gradient ascent/descent can get stuck in local maxima/minima and never reach the global spot.

Just as the names suggest, gradient ascent refers to finding the maximum objective (or cost) function value, while gradient descent focuses on finding the minimum. In practice, we generally only focus on finding the minimum since it has some computational benefits, even though mathematically it is the same. To turn a gradient ascent problem into a gradient descent problem, you simply multiply your objective function by \(-1\), or in other words, let \(f(x) = -g(x)\), and then apply the gradient descent algorithm instead. Gradient ascent focuses on moving with the gradient, while gradient descent moves in the opposite direction of the gradient.

The Gradient Descent Algorithm

Gradient Descent:

Let:

  • \(\beta_c\) = current objective function parameters
  • \(\beta_p\) = prev objective function parameters
  • \(\alpha\) = learning rate (\(\alpha \in [0, 1]\), usually)
  • \(f(\beta)\) = objective function
  • \(\sigma\) = tolerance for early stopping (\(\alpha\) is a small, positive constant)

Then:

  1. Initialize \(\beta_c\), \(\alpha\), \(\sigma\)
  2. For i in iterations:
    1. \(\beta_p := \beta_c\)
    2. For each \(\beta_{p_j}\) in {\(\beta_{p_1}\), \(\beta_{p_2}\), \(\beta_{p_n}\)}:
      1. \(\beta_{c_j} := \beta_{p_j} - \alpha \nabla f(\beta_p)\)
    3. \(\phi = \frac{\sum{(\beta_c - \beta_p)^2}}{n}\)
    4. If \(\phi <= \sigma\):
      1. break

Gradient descent can be a tricky algorithm to train because it can be easy to get stuck in local mimimums. The choice of learning rate (which tells the algorithm how large of steps to take when walking up/down the gradient) is very important. If you pick a rate that is too large, you will bounce around the minimum and never actually find it, or, you can bounce out of the global minimum into a different local minimum. If the learning rate is too small, convergence on the global minimum will be very, very slow (and it still doesnt mean it wont get stuck).

To diagnose how your gradient descent is going, it can be useful to calculate where the function is at in terms of cost. In our case, the cost is the log-likelihood, and so as we navigate the log-likelihood function, we want to print out the log-likelihood at each step (which in our case is actually negative log-likelihood, snce were doing gradient descent and want to minimize).

In the following R-example, we will see this in action. First, lets set up the basic parameters for the optimization.

# For reporducibility
set.seed(1234)

# Number of samples for our randomly generated data.
n <- 1000

# How difficult the problem is for the solver. This injects
# random noise into the target variable. (Higher is harder)
difficuly <- 3

# Create a dataset suitable for logistic regression
x0 <- rep(1, n)
x1 <- rnorm(n, 2, 0.1)
x2 <- rnorm(n, 5, 1)
x <- data.frame(b0=x0, b1=x1, b2=x2)
y <- -1*x0 + 2*x1 - 3*x2 + rnorm(n, 0, difficuly)
y_h <- ifelse(y < quantile(y, c(0.5)), 1, 0)

# Split into train/test sets
test_size <- 0.2
test_idx <- sample(1:n, as.integer(test_size*n))

x_train <- x[-test_idx, ]
y_train <- y_h[-test_idx]
x_test <- x[test_idx, ]
y_test <- y_h[test_idx]

Now that weve created our dataset for the problem, lets define the functions we will use to solve the problem.

# Partial derivative definition of the bernoulli likelihood, for each individual
# theta
PBD <- function(beta, xs, xi, ys, na.rm=FALSE) {
  mean((phat(as.matrix(xs) %*% as.matrix(beta)) - ys)*xi, na.rm=na.rm)
}

# Calculate Partial derivative for all thetas at once (using clever math)
# This is equivalent to `PBD`, but it uses matrices, so it will be faster
PBD_mat <- function(beta, xs, ys) {
  (t(xs) %*% (phat(as.matrix(xs) %*% as.matrix(beta)) - ys))/dim(xs)[1]
}

# Cost function for logistic regression, the negative log-likelihood
# for a set of bernoulli random variables
negloglike <- function(beta, xs, ys, use_mean=TRUE) {
  meanz <- phat(as.matrix(xs) %*% as.matrix(beta))
  if(use_mean) {
    loglike <- mean(dbinom(ys, 1, prob=meanz, log=TRUE))
  } else {
    loglike <- sum(dbinom(ys, 1, prob=meanz, log=TRUE))
  }
  return(-loglike)
}

# sigmoid function/transformation (i.e. the link function of the Generalized Linear Model)
phat  <- function(x) {
  return(1/(1+exp(-x)))
}

# A convenience function to calculate the target estimate
y_hat <- function(beta, xs, thres=0.5) {
  probs <- phat(as.matrix(xs) %*% as.matrix(beta))
  return(ifelse(probs>=thres, 1, 0))
}

The functions above provide all of the definitions we will need. Below, we perform the optimization. To start, we will initialize \(\beta\) (the regression coefficients) to be random normal values and the learning rate to 0.001. An early stopping criteria is implemented, such that the algorithm will stop if the mean square difference of the updated coefficients and the previous coefficients is between some precision (1e-12), or if the algorithm reaches the max number of iterations (100000), whichever comes first. For brevity, the algorithm will evaluate every 10000 iterations, but when doing this in an IDE or console, it is nice to make this update more frequently.

betas <- rnorm(dim(x)[2])
lr <- 0.001
precision <- 1e-12 
max_iters <- 100000
eval_i <- 10000

Gradient descent uses what is known as a simultaneous update, which means that each parameter is updated independently from the other parameters, or in other words, the parameters are all updated at the exact same time. In the commented section below, this is evidenced in the for loop (lines 5-8). After \(\beta_{1_{t+1}}\) is updated from {\(\beta_{1_{t}}, \beta_{2_{t}}, ...\), \(\beta_{2_{t+1}}\)} is also updated using {\(\beta_{1_{t}}, \beta_{2_{t}}, ...\)}, and not the newly available information \(\beta_{1_{t+1}}\) This is opposite of what would be called a sequential update.

Adaptive methods

Modern advances on gradient descent use more sphisticated means to tune the learning rate parameter for optimal performance, and an increased likelihood in finding the global minimum of a function. I wont cover all of them here, but I will introduce one of them: the Adam optimizer (Adaptive Moment Estimation). The Adam optimizer uses estimates of the gradient moments (average of past gradients and past square gradients) to adaptively tune the learning rate as the optimization proceeds. An initial choice of learning rate is still required, but the initial choice is less consequential in the Adam optimizer. The authors suggest using their defaults for the decay rates (usually defined as \(\beta_1\) and \(\beta_2\), but I will use \(\gamma\) to not confuse them with the coefficients).

Gradient Descent with Adam Optimizer:

Let:

  • \(\beta_c\) = current objective function parameters
  • \(\beta_p\) = prev objective function parameters
  • \(\alpha\) = learning rate (\(\alpha \in [0, 1]\), usually)
  • \(f(\beta)\) = objective function
  • \(\sigma\) = tolerance for early stopping (\(\alpha\) is a small, positive constant)
  • \(\gamma_1\) = Decay rate for first moment
  • \(\gamma_2\) = Decay rate for second moment
  • \(\epsilon\) = Small constant to avoid division by zero
  • \(m\) = First moment estimate
  • \(v\) = Second moment estimate

Then:

  1. Initialize \(\beta_c\), \(\alpha\), \(\sigma\)
  2. For i in iterations (1..n):
    1. \(\beta_p := \beta_c\)
    2. \(m_p := m_c\)
    3. \(v_p := v_c\)
    4. For each \(\beta_{p_j}\) in {\(\beta_{p_1}\), \(\beta_{p_2}\), \(\beta_{p_n}\)}:
      1. \(m_{c_j} := \gamma_1 m_{p_j} + (1 - \gamma_1) \nabla f(\beta)\)
      2. \(v_{c_j} := \gamma_2 v_{p_j} + (1 - \gamma_2) (\nabla f(\beta))^2\)
      3. \(\hat{m_{c_j}} = \frac{m_{c_j}}{1 - \gamma_{1}^{i}}\)
      4. \(\hat{v_{c_j}} = \frac{v_{c_j}}{1 - \gamma_{2}^{i}}\)
      5. \(\beta_{c_j} := \beta_{p_j} - \alpha \frac{\hat{m_{c_j}}}{\sqrt{\hat{v_{c_j}}} + \epsilon}\)
    5. \(\phi = \frac{\sum{(\beta_c - \beta_p)^2}}{n}\)
    6. If \(\phi <= \sigma\):
      1. break

GLMs and Neural Networks (NN)

A NN has many similarities to a linear model (though it is, by definition, non-linear). The formulation of a GLM can be summarised as \(Y = \sigma(X \beta)\) where \(\sigma\) is some link function. In neural networks, the link function is referred to as an activation function, but its purpose is similar, which is to conform the output of \(X \beta\) to some range. In the case of neural networks, the inputs pass through multiple activation functions (one for each layer) of the network. You could also think of a neural network as stacked GLMs, where the output of one GLM becomes the input of another GLM. These input/output combinations form a graph, and in a simple feed-forward network, the graph is straightforward, meaning it can be defined almost 100% in linear algebra. For a network with two hidden layers, it would look like this:

Let:

\(X_{m \times n}\) be the design matrix (bias included)

\(W_{1_{n \times a}}\) be the network weights for the first hidden layer

\(W_{2_{a \times b}}\) be the network weights for the second hidden layer

\(h_{b \times c}\) be the output layer (where c is either the number of classes, or number of classes - 1)

Then the feed-forward part of the network is simply:

\(X \beta_1 \beta_2 h = Y\)

such that the result of the matrix multiplication is:

$[m n] [n a] [a b] [b o] = m c $

Please note that the above equation is overly simplified, and does not account for the recursive nature of the activation functions. The above equations are just to give you a sense (in matrix algebra terms) of how the information between layers gets passed forward until it becomes an output. In the next paragraph, I will explain it min a more proper form.

Ill take this time right now to address bias (in terms of the coefficient/weight). In many online tutorials, papers, etc., bias is treated as its own term, for example \(Z = \sigma(XW + b)\). This same formulation is not uncommon for linear regression either, for example in \(Y = X \beta_{1..j} + \beta_0\), \(\beta_0\) is the bias term (though it has endless other names). However, this distinction is for convenience only, because just like you can write the linear regression equation as \(Y = X \beta\), you can write the layers of a NN as \(Z = \sigma(XW)\) and multiply the bias through as part of the design matrix. I say that formulating the problem as \(Z = \sigma(XW + b)\) is convenient because there may be application where you want to treat the bias separately from the design matrix, or leave it out entirely, etc. For most practical cases, however, it is generally part of the design matrix, and you can safely add a column of constant 1s to the beginning of the matrix to account for the bias.

Feed Forward Neural Network with Back Propagation

This algorithm provides a complete, explicit demonstration of a feed-forward neural network algorithm with backpropagation. This example uses two hidden layers, but the concepts can be applied to an arbitrary number of layers. The following assumes that \(f'(X)\) refers to the derivative of a function, \(X'\) refers to a matrix containing the direct or indirect result of such a derivative, \(X^T\) refers to the transpose of a matrix, and that \(A \times B\) means multiply element-wise. If either \(A\) or \(B\) is a matrix, then \(AB\) is assumed to be matrix multiplication.

A Feed-forward, Two hidden layer NN:

Let:

  • \(m\) = the number of observations
  • \(n\) = the number of features
  • \(h\) = the number of target columns (can be the number of classes, or the number of classes - 1)
  • \(X_{m \times n}\) = The design matrix (without bias)
  • \(Y_{m \times h}\) = The target variable
  • \(W_{1_{n \times a}}\) = The matrix of weights for the first hidden layer
  • \(W_{2_{a \times b}}\) = The matrix of weights for the second hidden layer
  • \(W_{h_{b \times h}}\) = The matrix of weights for the output layer
  • \(b_{1_{1 \times a}}\) = bias of the first hidden layer
  • \(b_{2_{1 \times b}}\) = bias of the second hidden layer
  • \(b_{h_{1 \times h}}\) = bias of the output layer
  • \(\sigma(X)_1\) = The activation function for the first layer
  • \(\sigma(X)_2\) = The activation function for the second layer
  • \(\sigma(X)_h\) = The output activation function
  • \(\sigma ' (X)_j\) = \(\frac{d \sigma}{d X} \sigma(X)\), where j is the respective layer
  • \(\eta\) = cost
  • \(\psi(Y, Z_h)\) = cost function, where \(Y\) is the true value, and \(Z_h\) is the prediction
  • \(\psi ' (Y, Z_h)\) = \(\frac{d \psi}{d Z_h} \psi(Y, Z_h)\)
  • \(\alpha\) = learning rate
  • \(T\) = total number of iterations

Then:

  1. Initialize \(W_1\), \(W_2\), \(W_h\), \(b_1\), \(b_2\), \(b_h\)
  2. For each \(i\) in {1, 2, T}:
    1. Perform Feed-forward operations
      1. \(Z_1 := \sigma(X W_1 + b_1)_1\)
      2. \(Z_2 := \sigma(Z_1 W_2 + b_2)_2\)
      3. \(Z_h := \sigma(Z_2 W_h + b_h)_h\)
    2. Compute Cost (for outputting progress, not algorithmically necessary)
      1. \(\eta = \psi(Y, Z_h)\)
    3. Perform Backpropagation
      1. \(Z_{h}' = \psi ' (Y, Z_h) \times \sigma_{h} ' (Z_2 W_h + b_h)\)
      2. \(W_h ' = \frac{Z_2^T Z_h '}{m}\)
      3. \(b_h ' = \frac{\sum Z_h'}{m}\) (sum the columns of \(Z_h'\))
      4. \(Z_{2}' = Z_{h}' W_h^T \times \sigma_2 ' (Z_2)\)
      5. \(W_2 ' = \frac{Z_1^T Z_2 '}{m}\)
      6. \(b_2 ' = \frac{\sum Z_2'}{m}\) (sum the columns of \(Z_2'\))
      7. \(Z_{1}' = Z_{2}' W_2^T \times \sigma_1 ' (Z_1)\)
      8. \(W_1 ' = \frac{X^T Z_1 '}{m}\)
      9. \(b_1 ' = \frac{\sum Z_1'}{m}\) (sum the columns of \(Z_1'\))
    4. Update weights and biases
      1. Perform gradient descent for each of the parameters calculated
        1. \(W_1 := W_1 - \alpha W_1 '\)
        2. \(b_1 := b_1 - \alpha b_1 '\)
        3. \(W_2 := W_2 - \alpha W_2 '\)
        4. \(b_2 := b_2 - \alpha b_2 '\)
        5. \(W_h := W_h - \alpha W_h '\)
        6. \(b_h := b_h - \alpha b_h '\)
    5. (Could implement an early stopping rule here)

Note that this example uses two hidden layers of arbitrary depth as an example. Once you understand whats happeneing here, you can generalize it to any size network. A generalized version of the algorithm can be found here

import numpy as np

def decision_boundary(x, thres=0.5):
    '''
    Divides probabilities `x` into two classes based on threshold value. Everything
    above the threshold is the positive class.
    '''
    y = np.zeros(shape=(len(x), 1))
    y[x >= thres] = 1
    return y


def relu(x):
    '''
    Retified Linear Unit. This activation function transforms the `x`
    values to 0 if they are negative, otherwise does nothing. This 
    function helps solve the vanishing gradient problem and is a popular 
    choice for hidden layers.
    '''
    x[x < 0] = 0
    return x


def sigmoid(x):
    '''
    The sigmoid function. It attempts to bound values that range from 
    -inf to inf between 0 and 1. As values move away from zero, they are 
    quickly pushed towards 0 or 1. This exponential property makes the sigmoid
    function a good choice for clasification and defining probabilites, because
    larger values quickly get sorted towards 0 and 1. For example, any `x` with
    and absolute value >= 2.197225 is automatically is assigned a probability >= 0.9.
    Also, between -1 and 1, the sigmoid function is approximately linear, so for
    small values of x, the activation function performs a linear transformation
    of the data, but for large values of x, it performs a non-linear transformation.
    Therefore, it is commonly used as an output activation function in neural networks,
    and a link function in statistics.
    
    The sigmoid function is also a popular choice for hidden layer activation, however, it
    suffers from the vanishing gradient problem. This is mainly due to it's expoential properties.
    '''
    return 1/(1 + np.exp(-x))


def relu_der(x):
    '''
    The derivative of the relu function with respect to x (usually the
    layer output or prediction value).
    
    Note: relu derivative is technically undefined at x=0, but
    tensorflow.nn.relu uses 0 for this special case, so that's what
    I do here
    '''
    dZ = x.copy()
    dZ[x <= 0] = 0
    dZ[x > 0] = 1
    return dZ


def sigmoid_der(x):
    '''
    The derivative of the sigmoid function. with respect to x
    (usually the target variable prediction)
    '''
    sigm = sigmoid(x)
    der = sigm * (1 - sigm)
    return der
    

def cross_entropy_1Dclass(y_true, y_pred, delta=1e-9):
    # While the definition varies across ML domains, Cross-entropy in general
    # is a method of measuring the difference of information between two
    # probability distributions (i.e. a way of measuring how similar they are).
    # In our context, cross entropy is the difference between our observed
    # values (which can be viewed as a probability distribution where every 
    # probability is either 0 or 1, since every value is known and thus has
    # an extreme probability) and the predicted values (which are actual 
    # probabilities since the true values are unknown). Since we are interested
    # in predicting values that follow a bernoulli distribution, the cross entropy
    # takes the form of the negative log-likelihood of the bernoulli distribution
    
    # With this cost function defined, Neural Networks are just performing
    # a more BA version of maximum likelihood estimation. A negative cost is
    # defined because maximizing the likelihood is the same as minimizing the
    # negative likelihood. 
    
    # m is just a scaling factor to make the computation easier in case the
    # gradients are really big

    # Compute the cross-entropy cost
    # To avoid log(0) errors
    y_pred[y_pred <= delta] = delta
    cost = -np.mean(xlogy(y_true, y_pred) + xlogy(1 - y_true, 1 - y_pred))
    return cost


def cross_entropy_1Dclass_der(y_true, y_pred, delta=1e-9):
    # The derivative of the cross-entropy function (i.e. derivative of the
    # error output) with respect to the predicted value (y_pred).
    # The value inside the brackets is the value of the derivative of the
    # log-likelihood for a bernoulli distribution. The negative is added
    # because of our definition of cross-entropy
    y_pred[y_pred <= delta] = delta
    y_pred[y_pred >= 1-delta] = 1-delta
    return -((y_true/y_pred) - (1-y_true)/(1-y_pred))

Now that we are familiar with the type of activations and cost functions well be using, we can begin to introduce the NN algorithm functions. Well start with the initialization function.

def initialize_wb(X, hidden_nodes=[6, 7, 10], scale=0.01, output_nodes=1,
                  random_state=None):
    '''
    Initializes the weights and biases. 
    
    X : The design matrix (without a bias/intercept term)
    hidden_nodes : A list defining how many hidden layers (the length of the list) 
                   and the depth of each layer (an integer). For example, [6] would
                   represent a single hidden layer with 6 nodes in the layer.
    scale : A float value to scale the initialized weights. Weights are initialized
            with a standard normal distribution. The scale adjusts the variance 
            of the weights
    output_nodes : An integer value specifying the number of nodes in the output
                   layer, which in the case of this code, is one less than the number
                   of classes (currently, this only supports a value of 1 (for binary
                   classification))
    random_state : An integer specifying the random seed. For reproducibility and debugging
    '''
    if random_state is not None:
        np.random.seed(random_state)
    
    # initialize empty dictionary for storing weights
    wb_list = {}
    xrows = X.shape[0]
    X = X.reshape(xrows, -1)
    prev_col = X.shape[1]

    if type(hidden_nodes) != list:
        raise TypeError("num_nodes must be a list")
    
    # add number of output nodes to the hidden_nodes list
    hidden_nodes.append(output_nodes)
    k = 0
    for i in hidden_nodes:
        cur_row = prev_col
        cur_col = i
        
        # Initialize weights to random normal values,
        # and bias to array of zeros
        W = np.random.normal(size=(cur_row, cur_col))*scale
        b = np.zeros(shape=(1, cur_col))
        
        # Store weights/biases in a dictionary
        wb_list["Weight" + str(k)] = W
        wb_list["bias" + str(k)] = b
        prev_col = cur_col
        k += 1

    return wb_list

Next, we will define forward propagation (feed-forward algorithm).

def forward_prop(X, actf, outf, wb):
    '''
    The feed forward algorithm. This function loops through each layer, and passes
    the activation of each layer forward to the next layer, like so:
    
    Z1 = actf(XW + b)
    Z2 = actf(ZW + b)
    etc...
    
    For the final layer, the output activation function is used instead
    of the hidden layer activation function. In theory, you could assign
    a different activation function for each layer. This implementation 
    keeps it simple and uses the same activation function for each hidden
    layer.
    
    `hz` is a matrix containing the activated values, i.e. actf(XW + b)
    `zs` is a dictionary containing the matrices BEFORE activation, for
        each layer. For example, zs['zs0'] = XW + b
    `hzs` is a dictionary containing the matrices AFTER activation, for
        each layer. For example, hzs['hzs0'] = actf(XW + b)
    
    X : The design matrix
    actf : the activation function for the hidden layers
    outf : the activation function for the output layer
    wb : a dictionary containing all the matrices of weights and biases
    '''
    
    # initialize the lists of matrices
    hz = X
    zs = {}
    hzs = {}
    
    # number hidden/output layers to iterate over
    ho_layers = int(len(wb)/2)
    for i in range(ho_layers):
        indx = str(i)
        
        # input of the previous layer multiplied by the weights of the current layer,
        # then add bias
        z = hz.dot(wb["Weight" + indx]) + wb["bias" + indx]
        
        # If final layer is the ouput layer, then apply the output activation
        # function instead of the hidden layer activation function
        if (i + 1) == ho_layers:
            hz = outf(z)
        else:
            hz = actf(z)
        
        # Store the activated/non-activated values for later use
        zs['z' + str(i)] = z
        hzs['hz' + str(i)] = hz
    return hz, zs, hzs

Next, the backpropagation algorithm. The back propagation algorithm can seem confusing, and every paper or text teaches this algorithm from a slightly different perspective, so it can be difficult to get a good understanding of it. I will attempt to explain in as simple, relevant, and important detail as possible. Along with my description, I have provided both code and an algorithm description that should help you piece it together as well. If the words arent doing it for you, I would step through the code in a debugger and just write down what is happening at each step.

In one sense, backpropagation is no more complicated than gradient descent. In fact, we will actually USE gradient descent to solve the neural network later on. In, gradient descent, we start at a random point. Based on that random point, we use the information in the gradients to decide where to go next to get to the minimum. In NNs, this is the same. Backpropagation is just the method we use to calculate the gradient to then update our parameters. We even use the same gradient descent formula (for a refresher, Im referring to \(\omega := \omega - \alpha \nabla f(\omega)\)). Instead of having \(\nabla f(\omega)\) defined in a straightforward formula, it must be calculated recursively in NNs using backpropagation.

Backpropagation takes advantage of two main concepts: recursion and what is known as the chain rule in calculus. The chain rule is relatively straightforward. Given a function \(f\) and a function \(g\), let \(h(x) = f(g(x))\). Then chain rule is simply that \(h'(x) = f'(g(x))*g'(x)\). In plain terms, this is just saying that the derivative of the nested function \(f(g(x))\) is equivalent to the derivative of the output function (\(f\)) multiplied by the derivative of the inside function (\(g\)). The input to \(f'\) remains as \(g(x)\). Another way to say this is to let \(y = g(x)\), then this can be rewritten as \(h'(x) = f'(y) g'(x)\) or \(h'(x) = f'(y) y'\).

Relating this back to NNs, to optimize our function, we want to find the gradient of a cost function, that used an activation function as an input, that also used an activation function as an input, and so on and so on So for a two layer NN, then chain is something like this:

\[h(W, X, b, Y) = \psi(Y, \sigma_{2}(\sigma_{1}(\sigma_{0}(XW_0 + b_0) W_1 + b_1) W_2 + b_2))\] Just like in our logistic regression gradient descent example, we are trying to calculate the gradient of the cost function with respect to the parameters. In this case, those parameters are the weights. In the logistic regression example, the calculation was straightforward because all of the weights came form the same layer. In the case of the NN, calculating the gradient using the chain rule for \(W_0\) will be longer than calculating the gradient for a weight closer to the output layer, such as \(W_2\).

Below is an example of the gradient (and applied chain rule) w.r.t. \(W_0\).

Let:

\(W_j\) = Weight of the jth Layer

\(b_j\) = The bias of the jth layer

\(\sigma_j\) = The activation function for the jth layer

\(\delta_j\) = The function to compute inputs x weights + bias for the jth layer

\(\psi\) = The cost function

\(X\) = The design matrix

\(Y\) = The target matrix/vector

Then:

\[h'(W_0 | X, b, Y) = \frac{d\psi}{d \sigma_2} \frac{d \sigma_2}{d \delta_2} \frac{d \delta_2}{d \sigma_1} \frac{d \sigma_1}{d \delta_1} \frac{d \delta_1}{d \sigma_0} \frac{d \sigma_0}{d \delta_0} \frac{d \delta_0}{d W_0} \psi(Y, \sigma_{2}(\sigma_{1}(\sigma_{0}(XW_0 + b_0) W_1 + b_1) W_2 + b_2))\] The expansion of this equation is too big to display here, but Ill get it started for you

\[h'(W_0 | X, b, Y) = \psi'(Y, \sigma_{2}(\sigma_{1}(\sigma_{0}(XW_0 + b_0) W_1 + b_1) W_2 + b_2)) \sigma_2 ' (\sigma_{1}(\sigma_{0}(XW_0 + b_0) W_1 + b_1)W_2 + b_2) W_2 \sigma_{1} ' (\sigma_{0}(XW_0 + b_0)W_1 + b_1) W_1 \sigma_{0} ' (XW_0 + b_0) X\]

Now, that equation looks really ugly, but it can actually be calculated very easily using recursion. As you may have guessed, if you were to iterate through each weight and calculate its respective derivative, it would be quite costly because you would end up calculating the same derivatives multiple times. Using the principle of recursion (which means to strategically use an output as an input to a new function), however, the calculations for all the weights and their derivatives can be done in one pass. To illustrate the function better, and show how the recursive part of the formula works, lets break the above equation into multiple pieces and recombine it.

Let:

\(Z_0 = XW_0 + b_0\)

\(A_0 = \sigma_0(Z_0)\)

\(Z_1 = A_0W_1 + b_1\)

\(A_1 = \sigma_1(Z_1)\)

\(Z_2 = A_1W_2 + b_2\)

\(A_2 = \psi(Z_2)\)

Then:

\[h'(W_0 | X, b, Y) = \psi ' (Y, A_2) \sigma_2 ' (Z_2) W_2 \sigma_1 ' (Z_1) W_1 \sigma_0 ' (Z_0) X\]

To make sure the concept was able to click, lets try another one. This time, well do w.r.t. \(W_2\).

\[h'(W_2 | X, b, Y) = \frac{d\psi}{d \sigma_2} \frac{d \sigma_2}{d \delta_2} \frac{d \delta_2}{d W_2} \psi(Y, \sigma_{2}(\sigma_{1}(\sigma_{0}(XW_0 + b_0) W_1 + b_1) W_2 + b_2))\] The equation above is the formulation of the chain rule, and the equation below is the direct expansion using the chain rule.

\[h'(W_2 | X, b, Y) = \psi'(Y, \sigma_{2}(\sigma_{1}(\sigma_{0}(XW_0 + b_0) W_1 + b_1) W_2 + b_2))\sigma_{2}'(\sigma_{1}(\sigma_{0}(XW_0 + b_0) W_1 + b_1) W_2 + b_2) Z_1\] The following equation is the same as the above, except in the recursive shorthand demonstrated earlier.

\[h'(W_2 | X, b, Y) = \psi ' (Y, A_2) \sigma_2 ' (Z_2) Z_1\]

By calculating the gradients in a backward fashion (starting from the output layer and working towards the input layer), backpropagation takes advantage of pre-computed gradients for gradient calculations down the line. This recursive nature is what makes back propagation so efficient.

def back_prop(X, Y, wb, zs, hzs, dactf, doutf, dcostf):
    '''
    Back Propagation algorithm. 
    
    '''
    # Some initializations for the loop
    m = X.shape[0]
    layers = int(len(wb)/2)
    dwdb = {}
    
    for i in range(layers-1, -1, -1):
        if i == layers-1:
            # If i is the output layer, use the derivative of the cost function
            # to start the chain rule
            dZ = dcostf(Y, hzs["hz" + str(i)]) * doutf(zs["z" + str(i)])
        else:
            # Otherwise, continue with the chain rule
            dZ = dZ.dot(wb["Weight" + str(i + 1)].T) * dactf(hzs["hz" + str(i)])
        if i == 0:
            # If i is the last layer, use the input data for the final calculation 
            A = X
        else:
            # Otherwise, use the next activation down the line
            A = hzs["hz" + str(i - 1)]
        
        # Gradient of the weight in layer i
        dW = np.dot(A.T, dZ)/m
        # gradient of the bias in layer i (note:
        # this could be represented in the weight
        # matrix as well, only seperated for convenience.
        # The dot product is performing the same sum
        # as below.
        dB = np.sum(dZ, axis=0, keepdims=True)/m
        
        # Store the gradients for updating later
        dwdb["Weight" + str(i)] = dW
        dwdb["bias" + str(i)] = dB
    return dwdb

Backpropagation and the innards of the NN

The hard part is over, and we can move on to running the code. But before we do, Im going to write out the entire process in algorithm form.

A Feed-forward NN General Algorithm:

Let:

  • \(m\) = the number of observations
  • \(n\) = the number of features
  • \(h\) = the number of target columns (can be the number of classes, or the number of classes - 1)
  • \(J\) = the total number of hidden and output layers
  • \(j\) = the current layer
  • \(X_{m \times n}\) = The design matrix (without bias)
  • \(Y_{m \times h}\) = The target variable
  • \(W_j\) = The jth matrix of weights
  • \(b_j\) = bias of the jth layer
  • \(\sigma(X)_j\) = The activation function for the jth layer
  • \(\sigma ' (X)_j\) = \(\frac{d \sigma}{d X} \sigma(X)\), where j is the respective layer
  • \(\eta\) = cost
  • \(\psi(Y, Z_h)\) = cost function, where \(Y\) is the true value, and \(Z_h\) is the prediction
  • \(\psi ' (Y, Z_h)\) = \(\frac{d \psi}{d Z_h} \psi(Y, Z_h)\)
  • \(\alpha\) = learning rate
  • \(T\) = total number of iterations

Then:

  1. Initialize \(W_J\), \(b_J\)
  2. For each \(i\) in {1, 2, T}:
    1. Perform Feed-forward operations
      1. \(Z_0 := X\)
      2. For \(j\) in \({1, 2, ..., J}\)
        1. \(Z_{j} := \sigma(Z_{j-1} W_j + b_j)_j\)
    2. Compute Cost (for outputting progress, not algorithmically necessary)
      1. \(\eta = \psi(Y, Z_J)\)
    3. Perform Backpropagation
      1. For \(j\) in \({J, ..., 2, 1}\)
        1. If layer is the output layer:
          1. \(Z_{j}' = \psi ' (Y, Z_j) \times \sigma_{j} ' (Z_{j-1} W_j + b_j)\)
        2. Else
          1. \(Z_{j}' = Z_{j+1}' W_{j+1}^T \times \sigma ' (Z_j)\)
        3. \(W_j ' = \frac{Z_{j-1}^T Z_j '}{m}\) (Gradient of the weight in layer j)
        4. \(b_j ' = \frac{\sum Z_j'}{m}\) (sum the columns of \(Z_j'\))
    4. Update weights and biases
      1. Perform gradient descent for each of the parameters calculated
        1. \(W_j := W_j - \alpha W_j '\)
        2. \(b_j := b_j - \alpha b_j '\)
    5. (Could implement an early stopping rule here)

Once again we have our choice of optimizer. Here, I have provided gradient descent and the Adam optimizer for comparison.

def gradient_descent(wb, dwdb, optimizer_args):
    for key in wb.keys():
        wb[key] = wb[key] - optimizer_args['learning_rate']*dwdb[key]
    return wb


def Adam(wb, dwdb, optimizer_args):
    for key in wb.keys():
        optimizer_args['mt'][key] = (optimizer_args['beta1']*optimizer_args['mt'][key] +
                                     (1-optimizer_args['beta1'])*dwdb[key])
        optimizer_args['vt'][key] = (optimizer_args['beta2']*optimizer_args['vt'][key] +
                                     (1-optimizer_args['beta2'])*(dwdb[key]**2))
        mhat = optimizer_args['mt'][key]/(1-(optimizer_args['beta1']**(optimizer_args['i']+1)))
        vhat = optimizer_args['vt'][key]/(1-(optimizer_args['beta2']**(optimizer_args['i']+1)))
        wb[key] = wb[key] - optimizer_args['learning_rate']*(mhat/(np.sqrt(vhat)+optimizer_args['eps']))
    return wb

Now, we put it all together.

def dnn(X, Y, actf, outf, costf, dactf, doutf, dcostf, optimizer,
        eval_size=1000, hidden_nodes=[5, 5], scale_weights=0.01,
        output_nodes=1, num_iterations=10000,
        optimizer_args={'learning_rate': 0.001},
        print_cost=True, random_state=None):
    
    # Initialize result arrays and weights
    costs = np.zeros(num_iterations)
    acc = np.zeros(num_iterations)
    
    # Load the function to update parameters based on string choice
    # of optimizer
    if optimizer == 'adam':
        update_parameters = Adam
    else:
        update_parameters = gradient_descent
    
    # At this point, everything we need to know about our network has been
    # defined, so let's initialize it with random numbers and draw it out
    wb = initialize_wb(X, hidden_nodes, scale_weights, output_nodes,
                       random_state=random_state)
    
    for i in range(num_iterations):

        # Forward propagation.
        (hz, zs, hzs) = forward_prop(X, actf, outf, wb)

        # Calculate cost
        cost = costf(y_true=Y, y_pred=hz)

        # Backpropagation.
        dwdb = back_prop(X, Y, wb, zs, hzs, dactf, doutf, dcostf)
        
        # Update parameters with optimizing function
        if optimizer == 'adam':
            # Adam has some extra parameters to keep track of, which is why
            # this is here
            if i == 0:
                # If it's the first iteration, generate the hyper-parameters
                # and load them into the optimizer_args dictionary
                optimizer_args['mt'] = {}
                optimizer_args['vt'] = {}
                for key in wb.keys():
                    optimizer_args['mt'][key] = np.zeros(wb[key].shape)
                    optimizer_args['vt'][key] = np.zeros(wb[key].shape)
            optimizer_args['i'] = i
            
        # Update the parameters
        wb = update_parameters(wb, dwdb, optimizer_args)
        
        # Store costs and accuracy
        costs[i] = cost
        acc[i] = (decision_boundary(hz) == Y).mean()

        # Print the cost every eval_size iterations
        if print_cost and i % eval_size == 0:
            print("Iteration %i: Cost: %f, Acc: %f" % (i, costs[i], acc[i]))
    return wb, hz, costs, acc
